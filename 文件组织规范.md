```

│   ├── OpenClaw/                 # OpenClaw 相关
│   └── 其他框架/

│   ├── 项目名称/
│   └── ...
├── Knowledge/                     # 知识库
│   ├── 编程语言/
│   ├── 框架技术/
│   └── 最佳实践/

└── OpenClaw/                      # OpenClaw 特定内容
```

## 命名规范

### 文件名

✅ **推荐**:
- 使用中文命名: `Claude-Code完整功能使用指南.md`
- 清晰描述内容: `Obsidian-AI自动标签系统部署指南.md`
- 使用连字符分隔英文单词: `Claude-Code`, `AI-Development`

❌ **避免**:
- 纯英文文件名 (除非是专有名词)
- 过长的文件名 (超过 50 字符)
- 特殊字符: `/ \ : * ? " < > |`

### 文件夹名

✅ **推荐**:
- 英文文件夹名 (便于路径引用): `AI-Development-Frameworks`
- 使用连字符或驼峰命名: `Claude-Code` 或 `ClaudeCode`
- 清晰的层级结构

## 内容分类

### AI-Development-Frameworks/
存放 AI 开发相关的框架、工具、指南

**子分类**:
- `Claude-Code/` - Claude Code 插件、配置、使用指南
- `OpenClaw/` - OpenClaw 架构、技能、配置
- `Cursor/` - Cursor 编辑器相关
- `其他工具/` - 其他 AI 开发工具

**文件类型**:
- 使用指南
- 配置文档
- 最佳实践
- 故障排查

### Projects/
存放具体项目的文档

**子分类**:
- 每个项目一个文件夹
- 项目名称使用中文或项目代号

**文件类型**:
- 项目规划
- 架构设计
- 开发日志
- 部署文档

### Knowledge/
存放通用知识和学习笔记

**子分类**:
- `编程语言/` - Python, Go, TypeScript 等
- `框架技术/` - React, Django, Next.js 等
- `最佳实践/` - 设计模式、代码规范等
- `工具使用/` - Git, Docker, Kubernetes 等

**文件类型**:
- 学习笔记
- 技术总结
- 代码片段
- 参考资料

### Archive/
存放不再活跃但需要保留的文档
