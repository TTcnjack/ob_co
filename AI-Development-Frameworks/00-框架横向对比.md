### 2. get-shit-done (GSD)

**定位**: A light-weight and powerful meta-prompting, context engineering and spec-driven development system

**核心特性**:
- ✅ **规范驱动开发** - 从需求到实现的完整流程
- ✅ **上下文工程** - 防止上下文腐烂(context rot)
- ✅ **多Agent编排** - 研究、规划、执行、验证分离
- ✅ **原子化Git提交** - 每个任务独立提交
- ✅ **Wave执行** - 基于依赖关系的并行执行
- ✅ **XML提示格式** - 结构化的任务定义

**工作流程**:
```
1. /gsd:new-project     → 初始化项目(问题→研究→需求→路线图)
2. /gsd:discuss-phase   → 讨论实现细节
3. /gsd:plan-phase      → 研究+规划+验证
4. /gsd:execute-phase   → Wave并行执行
5. /gsd:verify-work     → 人工验收测试
6. /gsd:complete-milestone → 完成里程碑
```

**技术架构**:
```
get-shit-done/
├── agents/           # 专业子代理
├── commands/gsd/     # GSD命令集
├── hooks/            # 钩子系统
├── scripts/          # 自动化脚本
└── docs/             # 用户指南
```

**优势**:
- 🏆 **最清晰的开发流程** - 6步工作流
- 🏆 **最好的上下文管理** - 防止质量退化
- 🏆 **最强的并行执行** - Wave系统
- 🏆 **最完整的验证循环** - 自动+人工验证

**劣势**:
- ⚠️ 流程较重,不适合小任务
- ⚠️ 需要严格遵循工作流
- ⚠️ 初始化成本高

### 3. ralph (Ralph Wiggum)

**定位**: Autonomous AI development loop with intelligent exit detection

**核心特性**:
- ✅ **自主循环** - 自动运行直到任务完成
- ✅ **智能退出检测** - 自动判断何时停止
- ✅ **多CLI支持** - Claude Code, Codex, OpenCode, Cursor
- ✅ **状态检查** - 实时监控进度
- ✅ **提示文件支持** - 从文件读取任务

**使用方式**:
```bash
# 基本用法
ralph "实现用户认证功能"

# 使用提示文件
ralph --prompt-file task.txt

# 检查状态
ralph --status
```

**技术架构**:
```
ralph/
├── Shell脚本        # 核心循环逻辑
├── 退出检测         # 智能停止机制
└── 状态管理         # 进度跟踪
```

**优势**:
- 🏆 **最简单的使用方式** - 一条命令搞定
- 🏆 **最强的自动化** - 无需人工干预
- 🏆 **最适合长时间运行** - 可以"过夜编码"

**劣势**:
- ⚠️ 缺乏精细控制
- ⚠️ 可能陷入无限循环
- ⚠️ 难以调试中间状态

/plugin marketplace add affaan-m/everything-claude-code
