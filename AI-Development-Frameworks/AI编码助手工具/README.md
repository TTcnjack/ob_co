## 工具列表

### 1. Ralph - 自动化开发循环执行器
**文件**: `Ralph自动化开发循环执行器.md`  
**GitHub**: https://github.com/frankbria/ralph-claude-code  
**类型**: Bash 应用程序  
**核心功能**: 自动化执行 Claude Code 的开发循环，智能监控和资源管理

**主要特性**:
- 自动化开发循环
- 电路断路器模式（Circuit Breaker）
- 速率限制（Rate Limiting）
- 实时监控（tmux 集成）
- 智能退出检测

**适合场景**:
- 长时间自主开发
- 资源优化
- 项目自动化
- 监控和调试

---

### 2. GSD (Get Shit Done) - 结构化开发生命周期
**文件**: `GSD结构化开发生命周期管理.md`  
**GitHub**: https://github.com/gsd-build/get-shit-done  
**类型**: 元提示系统（Meta-prompting System）  
**核心功能**: 为 AI 编码助手提供结构化的开发生命周期管理

**主要特性**:
- 阶段化开发生命周期（Discuss → Plan → Execute → Verify）
- Agent 编排
- 状态管理
- Git 集成
- 多模型支持

**适合场景**:
- 新项目开发
- Brownfield 开发
- Bug 修复
- 结构化项目管理

---

### 3. Superpowers - 技能驱动的开发工作流
**文件**: `Superpowers技能驱动开发工作流.md`  
**GitHub**: https://github.com/obra/superpowers  
**类型**: 技能系统（Skills System）  
**核心功能**: 为编码 Agent 提供完整的软件开发工作流

**主要特性**:
- 自动技能触发
- 核心工作流（头脑风暴 → Git 工作树 → 计划 → TDD → 审查）
- 技能库
- 多平台支持
- 子代理驱动开发

**适合场景**:
- 完整项目开发
- TDD 工作流
- 并行开发
- 代码审查
- 系统化调试

## 使用建议

**选择 Ralph 如果**:
- 需要长时间自主执行
- 关注资源优化
- 需要实时监控
- 使用 Claude Code

**选择 GSD 如果**:
- 需要结构化的开发流程
- 多阶段项目管理
- 需要验证和质量保证
- 团队协作

**选择 Superpowers 如果**:
- 需要完整的开发工作流
- 强制 TDD
- 多平台支持
- 自动化技能触发
